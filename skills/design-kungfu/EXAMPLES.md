# Design Kungfu Skill 使用示例

本文件演示 v2.6.0 的"扫描 → 按需追问 → 3 选 1 → 字段级合并"流程。所有示例遵循 SKILL.md 中定义的四条硬规则：

- 禁用 5 问清单（"产品类型 / 目标用户 / 品牌调性 / 主要设备 / 特殊要求"）
- 禁用 ASCII 盒子框（`┌──┐`、`│`、`├──┤`）
- 禁止向用户展示数字评分（"93/100"、"40/40"、"★★★★★"）
- 禁止在用户可见文案中出现开发者术语（`Geist`、`tracking-tight`、`hex`、`token`、`WCAG` 等）

---

## 示例 1：低覆盖度（"帮我做个网站"）

**用户输入**：

> 帮我做个网站

**AI 扫描结果**（内部，不展示给用户）：

- 产品类型：❌ 未给
- 品牌调性：❌ 未给
- 视觉约束：❌ 未给

0 维明确 → 走一次口语化追问（禁止 5 问清单）。

**AI 回复**：

> 这个项目是做什么的？想要什么感觉——商务点 / 活泼点 / 极简点 / 大胆点，随便说一个都行。

**用户继续**：

> 做一个小团队的官网，我们是做独立游戏的，想要有点个性，不要太严肃。

**AI 扫描结果**（内部更新）：

- 产品类型：✅ 官网 / 独立工作室
- 品牌调性：✅ 个性、有活力、不严肃
- 视觉约束：❌ 未给

≥ 2 维明确 → **跳过追问，直接进入第二步内部打分**。

**AI 回复**：

> 给你挑了 3 个最合适的风格，选一个我就开始做：
>
> 1️⃣ Neo-Brutalist Playful
>    特点：粗黑边框、鲜艳色块、有点故意不规整的反套路感
>    为什么适合你：独立游戏工作室最想避开"千篇一律"，这个风格自带个性
>
> 2️⃣ Bento Grid
>    特点：大小卡片拼图式布局，像 iOS 小组件那种
>    为什么适合你：展示多款游戏作品特别合适，一眼看完全貌
>
> 3️⃣ Modern Gradient
>    特点：大面积渐变色 + 现代无衬线字体，年轻有活力
>    为什么适合你：比 Neo-Brutalist 温和一点，但依然不严肃
>
> 回复 1 / 2 / 3 或直接说风格名就行。

**用户选定**：

> 1

**AI 后续**：读取 `styles/neo-brutalist-playful.md`，按 tokens 生成官网落地页代码。

**流程要点**：

- 首轮不塞 5 问清单，只用一句大白话把最关键的两件事问出来
- 用户补充后，两个维度明确即进入打分，不再追问第三维
- 候选卡片没有任何数字评分、ASCII 盒子、Tailwind class

## 示例 2：中覆盖度 + 参考品牌（字段级合并）

**用户输入**：

> 我要做一个团队项目管理 SaaS 的落地页，风格参考 Linear，但主色想用深蓝，不要 Linear 的紫色。目标是给中大型团队用的，要显得专业。

**AI 扫描结果**（内部）：

- 产品类型：✅ SaaS / 项目管理
- 品牌调性：✅ 专业、冷静
- 视觉约束：✅ 参考品牌 Linear（软性提及，非指令性点名）；主色深蓝（方向性描述，非精确值）

维度全命中，但：

- "参考 Linear" ≠ "用 Linear 风格做"——属软性提及，不触发快速通道 #1
- "深蓝"是方向性描述而非精确值（`#0A2540` 这种），不计入快速通道 #2 的字段数
- 只有 1 个视觉字段被方向性锁定 → 不满足 ≥ 3 项精确值

→ 走第三步正常推荐流程，但 Linear 会以高分出现在候选里。

**AI 回复**：

> 给你挑了 3 个最合适的风格，选一个我就开始做：
>
> 1️⃣ Linear 风格
>    特点：冷静现代、深色配强调色、项目管理软件的标杆调性
>    为什么适合你：你已经点名参考 Linear，主色换深蓝也完全对得上这套骨架
>
> 2️⃣ Vercel 风格
>    特点：黑白为主、大号字号、充足留白，技术味重
>    为什么适合你：中大型团队看到一眼就觉得"很专业"的另一种选择
>
> 3️⃣ Stripe 风格
>    特点：深蓝为底、精致渐变，金融级稳重
>    为什么适合你：主色本来就是深蓝系，上手即用，最省事
>
> 回复 1 / 2 / 3 或直接说风格名就行。

**用户选定**：

> 1

**AI 进入第五步 —— 字段级合并**：

- 读取 `references/linear-app.md`（优先）+ `styles/linear-style.md`
- 字段级合并规则：
  - **用户未指定的字段**（排版、间距、圆角、阴影、动效、字体层级、组件结构）→ 全部沿用 Linear tokens
  - **用户显式指定的字段**（强调色）→ 把 Linear 原本的紫色 `#5E6AD2` 替换为深蓝（AI 根据"深蓝"选择合理值，如 `#1E3A8A`，并在代码注释里标明理由）

**关键代码片段**（节选）：

```html
<!-- 字段级合并：Linear 骨架 + 深蓝强调色覆盖 -->
<section class="bg-[#08090A] text-white min-h-screen">
  <div class="max-w-6xl mx-auto px-6 py-24">
    <h1 class="text-6xl font-semibold tracking-tight leading-[1.05]">
      The issue tracker built for
      <span class="text-[#1E3A8A]">high-performance</span> teams
      <!-- ↑ 原 Linear 紫色 #5E6AD2 被用户指定的深蓝替换 -->
    </h1>
    <!-- 其余排版、间距、字号全部沿用 Linear tokens -->
  </div>
</section>
```

**流程要点**：

- "参考 X" 是软性提及 → 进第三步，但把 X 放进候选里让用户再确认一次
- 字段级合并：只覆盖用户显式说过的那一项，其他全部沿用参考品牌 tokens
- 生成代码里可以出现 `#1E3A8A` `tracking-tight` 这类开发者术语，因为它们在"第五步代码输出"里，不受"用户可见文案禁用开发者术语"规则约束

## 示例 3：中覆盖度 + 无参考品牌

**用户输入**：

> 做一个给自由职业者用的时间追踪工具，网页端就行。希望界面清爽、不打扰人，专注感强一点。

**AI 扫描结果**（内部）：

- 产品类型：✅ 工具类 / 生产力应用
- 品牌调性：✅ 清爽、不打扰、专注
- 视觉约束：❌ 未给（"清爽"、"不打扰"都是方向性描述，且无具体色值 / 字体 / 参考品牌）

≥ 2 维明确 → 跳过追问，直接进入第二步。无快速通道命中 → 走第三步。

**AI 回复**：

> 给你挑了 3 个最合适的风格，选一个我就开始做：
>
> 1️⃣ Minimalist Flat
>    特点：纯色块、直角设计、零装饰，像瑞士设计手册那种克制
>    为什么适合你：专注工具最怕花哨，这个风格让界面彻底"消失"
>
> 2️⃣ Cal.com 风格
>    特点：清爽白底、轻量卡片、柔和强调色
>    为什么适合你：调性本来就是"不打扰的生产力工具"，天然契合
>
> 3️⃣ Dashboard Layout
>    特点：左侧窄边栏 + 主区域大数据视图的经典结构
>    为什么适合你：时间追踪最终要看数据，这套布局直接解决展示问题
>
> 回复 1 / 2 / 3 或直接说风格名就行。

**用户选定**：

> 2

**AI 进入第五步**：读取 `styles/cal-com-style.md`，按 tokens 生成网页端界面。所有字段均使用 Cal.com 默认 tokens（用户未指定任何视觉字段）。

**流程要点**：

- 没有参考品牌的情况下，候选应覆盖不同的解题思路：视觉风格（Minimalist Flat）、产品调性品牌（Cal.com）、布局结构（Dashboard Layout）——避免 3 个候选同质化
- 用户说的"清爽、不打扰、专注"映射到 `brand_moods[极简]` + `audiences[创意人士/自由职业者]`
- 无任何精确视觉字段 → 不触发快速通道 #2，正常展示 3 个候选

## 示例 4：高覆盖度（触发快速通道 #2）

**用户输入**：

> 帮我做 MeooDream 壁纸生成器的落地页。主色用 `#00F2FF` 青色，背景 `#0A0A0A` 纯黑。字体用 Manrope。卡片圆角 `rounded-3xl`，悬停时 `scale(1.05)` 配 `200ms ease-out`。布局用三栏卡片网格，1200px 居中，配一个顶部大 hero 区。整体赛博朋克感 + 梦幻。

**AI 扫描结果**（内部）：

- 产品类型：✅ 工具类 / AI 产品
- 品牌调性：✅ 赛博朋克 + 梦幻
- 视觉约束：✅ **精确值锁定 5 项**
  1. 主色 `#00F2FF` ✓
  2. 背景色 `#0A0A0A` ✓
  3. 字体 `Manrope` ✓
  4. 圆角 `rounded-3xl` ✓
  5. 动效 `scale(1.05)` + `200ms ease-out` ✓
  6. 布局 "三栏卡片网格，1200px 居中"（精确到栏数 + 容器宽度）✓

精确值 ≥ 3 项 → **触发快速通道 #2**，跳过第三步候选展示。

内部打分仍进行：`brand_moods[大胆/科技感]` + `product_types[AI产品]` → Top 1 = Cyberpunk Neon（或 Glassmorphism，取决于具体打分）。仅用作**组件骨架参考**。

**AI 回复**（不展示候选，直接给到下一步确认）：

> 看到你把色值、字体、圆角、动效、布局都给到精确值了，我直接按你给的参数做。风格骨架我会参考 Cyberpunk Neon（赛博朋克霓虹），它的光效和发光边框风格跟你要的调性最搭。你给的所有字段我都不会动，只补上你没提的那些（比如次要文字颜色、间距层级、按钮 hover 发光等）。
>
> 现在开始生成落地页，稍等。

**AI 进入第五步**：读取 `styles/cyberpunk-neon.md`，按 Override Chain 字段级合并：

| 字段 | 来源 | 值 |
|------|------|----|
| 主色 | **用户显式** | `#00F2FF` |
| 背景 | **用户显式** | `#0A0A0A` |
| 字体 | **用户显式** | `Manrope` |
| 卡片圆角 | **用户显式** | `rounded-3xl` |
| 卡片 hover 动效 | **用户显式** | `scale(1.05)` `200ms ease-out` |
| 主布局 | **用户显式** | 三栏 1200px 居中 |
| 次要文字色 | Cyberpunk Neon 默认 | `text-cyan-100/70` |
| 按钮发光阴影 | Cyberpunk Neon 默认 | `shadow-[0_0_24px_#00F2FF66]` |
| hero 区装饰 | Cyberpunk Neon 默认 | 径向渐变光晕 |

**流程要点**：

- 触发条件复核：5 项精确值 ≥ 3 项 ✅；都是"可直接写进代码的值"而非"青色""无衬线"这类方向性描述 ✅
- 跳过第三步后，**不代表不告诉用户在做什么**——一句话点明"我参考了 Cyberpunk Neon 作为骨架"，让用户有机会拦截
- 任何用户显式给过的字段，即使 Cyberpunk Neon 的 `forbidden` 列表里禁止（假设禁止某种圆角值），也以用户为准（SKILL.md AI 输出规则第二条）

## 四个示例的流程对照表

| 示例 | 产品类型 | 品牌调性 | 视觉约束 | 追问 | 候选展示 | 快速通道 |
|------|---------|---------|---------|------|---------|---------|
| 1 官网 | ❌ → ✅（追问后） | ❌ → ✅（追问后） | ❌ | 1 次 | ✅ | — |
| 2 SaaS 参考 Linear | ✅ | ✅ | ✅（软性 + 方向性）| 0 次 | ✅ | — |
| 3 时间追踪 | ✅ | ✅ | ❌ | 0 次 | ✅ | — |
| 4 MeooDream | ✅ | ✅ | ✅（5 项精确值）| 0 次 | ❌（跳过）| #2 |

**关键分岔点**：

- 追问与否 = 扫描后明确维度数（0 维追 1 次，1 维追 1 次缺的那项，≥2 维不追）
- 快速通道 #1 = 用户用**指令性措辞**（"用 X"、"按 X 来"、"make it like X"）点名风格
- 快速通道 #2 = 用户以**精确值**锁定 ≥ 3 项关键视觉字段
- 软性提及（"参考 X"）或方向性描述（"深蓝"、"大圆角"）**都不触发快速通道**

## 反面示例（v2.5.0 及之前的格式，已废弃）

以下格式**禁止再出现**。保留作为反面教材，让 AI 记住哪些是老模式的遗留陷阱。

### ❌ 禁用：5 问清单

```
🔍 让我了解一下你的项目：

1. 产品类型：SaaS / 电商 / Fintech / 教育科技 / 开发者工具 / 其他
2. 目标用户：企业 / 消费者 / 开发者 / 创意人士
3. 品牌调性：专业 / 活泼 / 奢华 / 极简 / 大胆
4. 主要设备：桌面端 / 移动端 / 全平台
5. 特殊要求：无障碍优先 / 深色模式 / 性能优先
```

**问题**：一次把 5 件事问完，小白用户看到就关页面。替换成 0-1 次口语化追问。

### ❌ 禁用：ASCII 盒子 + 数字评分

```
┌─────────────────────────────────────────────────────────────┐
│  🥋 DESIGN KUNGFU 设计推荐                                  │
├─────────────────────────────────────────────────────────────┤
│  🏆 最佳匹配                                                │
│  Style: Vercel Style                                        │
│  匹配度: 93/100                                              │
│  理由: 黑白 Geist 字体 + 极简                                │
│                                                             │
│  📊 评分                                                    │
│  • 产品类型: 40/40   • 品牌调性: 33/35   • 目标受众: 20/25  │
└─────────────────────────────────────────────────────────────┘
```

**问题**：ASCII 盒子在用户端经常渲染错位；93/100 这种评分对非开发者是视觉噪音；`Geist` 是开发者术语（具体字体名称），用户看不懂。替换成口语化的 `特点 / 为什么适合你` 卡片。

### ❌ 禁用：复读式确认

```
用户: 我要做一个 SaaS 项目管理工具
AI: 好的！你是要做一个 SaaS 项目管理工具对吧？那我问几个问题：
    产品类型：SaaS ✓
    目标用户：？
    品牌调性：？
```

**问题**：用户已经说了"SaaS 项目管理工具"，再复述一遍是浪费。直接扫描、进打分、需要补什么只补一次。

### ❌ 禁用：在用户可见文案里写开发者术语

```
AI: 推荐你用 Vercel 风格：Geist Sans 字体 + tracking-tight + bg-black text-white
    配合 border-zinc-800 分割线，符合 WCAG 2.1 AA 对比度要求。
```

**问题**：`Geist Sans` `tracking-tight` `bg-black` `WCAG 2.1 AA` 全是开发者术语，小白用户看得一头雾水。这些词允许出现在**第五步生成的代码里**，但不能出现在第三步的候选卡片、第一步的追问、或任何和用户对话的文案里。

**版本对应**：本文档对应 SKILL.md v2.6.0，与 v2.5.0 及之前的示例格式**完全不兼容**。
