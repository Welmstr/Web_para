# Component Patterns 组件设计模式

## 概述

本文档提供常见 UI 组件的设计模式和最佳实践，帮助创建一致、可用、美观的组件。

## 按钮 Button

### 变体
- **Primary**：主要操作，使用主色
- **Secondary**：次要操作，使用边框或浅色背景
- **Ghost**：透明背景，悬停时显示背景
- **Danger**：危险操作，使用红色

### 尺寸
- **Small**：32px 高度，用于紧凑空间
- **Medium**：40px 高度，默认尺寸
- **Large**：48px 高度，用于强调

### 状态
- Default、Hover、Active、Disabled、Loading

## 卡片 Card

### 结构
```
┌─────────────────────────┐
│  [Header/图片区域]      │
├─────────────────────────┤
│  Title                  │
│  Description text...    │
├─────────────────────────┤
│  [Footer/操作区]        │
└─────────────────────────┘
```

### 变体
- **Default**：标准卡片，带阴影
- **Outlined**：边框卡片，无阴影
- **Elevated**：悬浮效果，强阴影

## 模态框 Modal

### 结构
- **Overlay**：半透明遮罩层
- **Container**：居中容器，固定最大宽度
- **Header**：标题和关闭按钮
- **Body**：内容区域
- **Footer**：操作按钮

### 行为
- 点击遮罩层关闭（可选）
- ESC 键关闭
- 打开时禁止背景滚动
- 聚焦陷阱（Focus Trap）

## 表单输入 Input

### 结构
```
[Label]
[Input Field          ]
[Helper text / Error  ]
```

### 状态
- Default、Focus、Error、Disabled、Read-only

### 图标
- 前缀图标：用于提示输入类型
- 后缀图标：用于操作（清除、密码切换）

## 下拉菜单 Dropdown

### 触发方式
- 点击触发
- 悬停触发（谨慎使用）

### 位置
- 自动计算最佳位置
- 支持上、下、左、右四个方向

### 内容
- 菜单项列表
- 分割线
- 分组标题

## 导航 Navigation

### 顶部导航
- Logo 区域
- 主导航链接
- 用户操作区

### 侧边导航
- 可折叠分组
- 当前项高亮
- 图标 + 文字

### 面包屑
- 层级路径展示
- 可点击跳转
- 当前页不可点击

## 表格 Table

### 结构
- 表头：列标题，可排序
- 表体：数据行
- 表尾：汇总信息（可选）

### 功能
- 排序
- 筛选
- 分页
- 行选择
- 空状态

## 标签页 Tabs

### 类型
- **Line**：下划线样式
- **Pill**：胶囊按钮样式
- **Card**：卡片式

### 行为
- 点击切换
- 支持禁用某些标签
- 可滚动（标签过多时）

## 提示 Tooltip

### 触发
- 悬停显示
- 延迟显示（避免闪烁）

### 位置
- 自动调整避免溢出
- 支持 8 个方向

### 内容
- 纯文本
- 富文本（谨慎使用）

## 警告 Alert

### 类型
- **Info**：信息提示，蓝色
- **Success**：成功提示，绿色
- **Warning**：警告提示，黄色
- **Error**：错误提示，红色

### 结构
- 图标
- 标题（可选）
- 描述文字
- 关闭按钮（可选）

## 加载 Loading

### 类型
- **Spinner**：旋转圆圈
- **Skeleton**：骨架屏
- **Progress**：进度条

### 使用场景
- Spinner：小区域加载
- Skeleton：内容预占位
- Progress：可量化进度
