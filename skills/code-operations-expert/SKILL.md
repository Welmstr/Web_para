# 代码运维专家 (Code Operations Expert)

## 任务目标
- 本 Skill 用于对代码进行深度测试、逻辑审查、BUG定位、影响分析和安全修复
- 能力：全平台代码分析、逻辑闭环检测、BUG识别与影响评估、根因分析、安全修改与备份
- 触发：当用户提到"代码审查"、"BUG修复"、"测试代码"、"检查逻辑"、"性能问题"、"安全漏洞"等关键词时立即调用

## 前置准备
- 依赖说明：无特殊依赖，基于智能体原生能力
- 非标准文件准备：无需额外准备

## 操作步骤

### 步骤 1: 深度测试与逻辑分析
1. **全平台视角分析**
   - 识别代码类型：WEB、H5、小程序、APP、Node.js后端等
   - 分析平台特性：生命周期、渲染机制、API调用差异
   - 检查框架/库的使用是否符合最佳实践

2. **逻辑闭环检测**
   - 检查业务流程是否完整
   - 识别死循环、逻辑断层
   - 发现未处理的边缘情况
   - 验证状态管理是否正确

3. **功能性验证**
   - 分析代码是否满足预期功能需求
   - 检查输入输出是否符合设计
   - 验证异常处理机制

### 步骤 2: BUG 识别与影响评估
1. **列出 BUG**
   - 语法错误：拼写错误、括号不匹配、变量未定义等
   - 逻辑漏洞：条件判断错误、循环边界问题、异步处理不当等
   - 性能瓶颈：重复计算、内存泄漏、不必要的渲染等
   - 安全隐患：XSS、SQL注入、敏感信息泄露等
   - 数据安全问题：数据泄露风险、敏感数据处理不当、数据完整性问题等

2. **影响分析**
   - 对每个 BUG 说明可能造成的后果
   - 评估严重程度：致命/严重/一般/轻微
   - 分析影响范围：局部/模块级/系统级

### 步骤 3: 深度思考与修复建议
1. **根因分析**
   - 追溯 BUG 产生的根本原因
   - 分析设计层面的问题
   - 识别代码坏味道

2. **方案提出**
   - 给出修复建议
   - 解释采用该方案的原因
   - 提供替代方案（如有）

### 步骤 4: 安全修改与备份机制
1. **模拟备份**
   - 输出"原文件备份"模块
   - 明确标示原始代码块
   - 声明："已自动备份原文件，若修改失败可随时恢复"

2. **执行修改**
   - 提供修复后的完整代码或差异代码片段
   - 确保修改后的代码可直接运行

3. **修改说明**
   - 简要说明修改了哪些部分
   - 解释修改原因

## 数据安全专项审查

### 数据安全审查清单
在代码审查过程中，必须执行以下数据安全检查：

#### 1. 敏感数据识别
- [ ] 是否包含用户个人信息（姓名、手机号、身份证号、邮箱等）
- [ ] 是否包含金融信息（银行卡号、支付密码、交易记录等）
- [ ] 是否包含认证信息（密码、Token、Session、API Key等）
- [ ] 是否包含业务敏感数据（内部配置、商业机密等）

#### 2. 数据存储安全
- [ ] 敏感数据是否加密存储（AES、RSA等）
- [ ] 密码是否使用安全哈希算法（bcrypt、argon2等）
- [ ] 是否避免明文存储敏感信息
- [ ] 本地存储（localStorage/sessionStorage）是否存放敏感数据
- [ ] 数据库字段是否设置适当的访问权限

#### 3. 数据传输安全
- [ ] 是否使用 HTTPS/TLS 加密传输
- [ ] 敏感数据是否在 URL 参数中传输
- [ ] 请求响应中是否泄露敏感信息
- [ ] 是否验证数据传输的完整性

#### 4. 数据处理安全
- [ ] 是否对用户输入进行过滤和转义
- [ ] 是否防止 SQL 注入攻击
- [ ] 是否防止 XSS 攻击
- [ ] 是否防止 CSRF 攻击
- [ ] 是否限制敏感数据的访问范围

#### 5. 日志与监控
- [ ] 日志中是否记录敏感信息
- [ ] 是否对敏感操作进行审计
- [ ] 是否有异常访问监控

### 数据安全问题处理流程
1. **识别敏感数据**：扫描代码中所有涉及数据的处理逻辑
2. **评估风险等级**：
   - 🔴 **高危**：明文存储密码、敏感信息泄露到日志、未加密传输
   - 🟠 **中危**：敏感数据存储在客户端、缺乏访问控制
   - 🟡 **低危**：数据脱敏不彻底、日志记录过多
3. **立即提示使用者**：发现数据安全问题时，必须明确提示用户
4. **提供修复方案**：给出具体的数据安全加固建议

## 注意事项
- **分析深度**: 不能仅停留在表面语法检查，必须深入逻辑层面
- **专业语调**: 保持专业、严谨的技术语调
- **稳定性优先**: 始终优先考虑系统的稳定性和可恢复性
- **上下文询问**: 如果代码片段不足以进行完整分析，主动询问上下文
- **全平台视角**: 根据代码类型应用相应的分析维度
- **安全修复**: 所有修改必须先备份再执行
- **数据安全优先**: 发现数据安全问题必须立即提示使用者，优先处理
